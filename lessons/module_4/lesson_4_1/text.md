<b>🧩 Урок 4.1. Создание и вызов функций</b>

<i>Функции — один из важнейших инструментов программирования. Они позволяют разбивать программу на небольшие части, избегать повторения кода и делать программы более понятными.</i>

━━━━━━━━━━━
<b>ЧАСТЬ 1. Что такое функция</b>

Представьте, что вам нужно много раз выводить одно и то же сообщение.

Без функций:

<pre><code class="language-python">print("Добро пожаловать!")
print("Добро пожаловать!")
print("Добро пожаловать!")</code></pre>

Код начинает повторяться.

Функция позволяет сохранить набор действий под именем и выполнять его в любой момент.

Функцию можно представить как отдельную мини-программу внутри вашей программы.

━━━━━━━━━━━
<b>Зачем нужны функции</b>

Функции помогают:

• не дублировать код

• делать программу понятнее

• легче исправлять ошибки

• разбивать большие задачи на маленькие

Практически любая крупная программа состоит из десятков, сотен и даже тысяч функций.

━━━━━━━━━━━
<b>ЧАСТЬ 2. Создание функции</b>

Для создания функции используется ключевое слово:

<pre><code class="language-python">def</code></pre>

Общий синтаксис:

<pre><code class="language-python">def имя_функции():
    код функции</code></pre>

Пример:

<pre><code class="language-python">def hello():
    print("Привет!")</code></pre>

Разберём по частям:

• <code>def</code> — создание функции

• <code>hello</code> — имя функции

• <code>()</code> — скобки функции

• <code>:</code> — начало блока кода

• отступ определяет тело функции

После запуска программы функция создаётся, но ещё не выполняется.

━━━━━━━━━━━
<b>ЧАСТЬ 3. Вызов функции</b>

Чтобы функция выполнилась, её нужно вызвать.

Для этого используется её имя со скобками.

<pre><code class="language-python">def hello():
    print("Привет!")

hello()</code></pre>

Результат:

<pre><code>Привет!</code></pre>

Сначала Python создаёт функцию.

Затем встречает:

<pre><code class="language-python">hello()</code></pre>

и выполняет код внутри неё.

━━━━━━━━━━━
<b>Несколько вызовов одной функции</b>

Одну функцию можно вызывать много раз.

<pre><code class="language-python">def hello():
    print("Привет!")

hello()
hello()
hello()</code></pre>

Результат:

<pre><code>Привет!
Привет!
Привет!</code></pre>

Функция написана один раз, а выполняется сколько угодно раз.

В этом её главное преимущество.

━━━━━━━━━━━
<b>ЧАСТЬ 4. Несколько функций в программе</b>

В программе может быть много функций.

Например:

<pre><code class="language-python">def hello():
    print("Привет!")

def bye():
    print("Пока!")

hello()
bye()</code></pre>

Результат:

<pre><code>Привет!
Пока!</code></pre>

Каждая функция отвечает за свою задачу.

Так программы становятся более организованными.

━━━━━━━━━━━
<b>ЧАСТЬ 5. Что такое return</b>

Очень важно понимать разницу между:

<pre><code class="language-python">print()</code></pre>

и

<pre><code class="language-python">return</code></pre>

Новички постоянно их путают.

Сначала пример с print:

<pre><code class="language-python">def get_number():
    print(10)

get_number()</code></pre>

Результат:

<pre><code>10</code></pre>

Число просто вывелось на экран.

━━━━━━━━━━━
<b>Возврат значения через return</b>

Теперь используем return:

<pre><code class="language-python">def get_number():
    return 10</code></pre>

Вызов:

<pre><code class="language-python">result = get_number()

print(result)</code></pre>

Результат:

<pre><code>10</code></pre>

Здесь число не просто вывелось.

Оно вернулось обратно в программу и сохранилось в переменную.

━━━━━━━━━━━
<b>ЧАСТЬ 6. Разница между print и return</b>

Сравним.

Вариант через print:

<pre><code class="language-python">def get_number():
    print(10)

x = get_number()

print(x)</code></pre>

Результат:

<pre><code>10
None</code></pre>

Почему появился None?

Потому что функция ничего не вернула.

━━━━━━━━━━━

Вариант через return:

<pre><code class="language-python">def get_number():
    return 10

x = get_number()

print(x)</code></pre>

Результат:

<pre><code>10</code></pre>

Теперь значение успешно сохранилось в переменную.

━━━━━━━━━━━
<b>Запомните правило</b>

<b>print()</b>

• показывает информацию пользователю

<b>return</b>

• возвращает результат программе

Если результат понадобится дальше в коде — используйте return.

━━━━━━━━━━━
<b>ЧАСТЬ 7. Возврат результата вычислений</b>

Функция может возвращать не только готовое значение, но и результат вычисления.

Пример:

<pre><code class="language-python">def get_sum():
    return 5 + 3</code></pre>

Вызов:

<pre><code class="language-python">result = get_sum()

print(result)</code></pre>

Результат:

<pre><code>8</code></pre>

Python сначала вычисляет:

<pre><code class="language-python">5 + 3</code></pre>

а затем возвращает результат.

━━━━━━━━━━━
<b>Функцию можно использовать прямо внутри print</b>

<pre><code class="language-python">def get_sum():
    return 5 + 3

print(get_sum())</code></pre>

Результат:

<pre><code>8</code></pre>

Такой стиль встречается очень часто.

━━━━━━━━━━━
<b>ЧАСТЬ 8. Функции могут вызывать другие функции</b>

Одна функция может использовать другую.

Пример:

<pre><code class="language-python">def hello():
    print("Привет!")

def start():
    hello()
    print("Программа запущена")

start()</code></pre>

Результат:

<pre><code>Привет!
Программа запущена</code></pre>

Именно так строятся большие проекты.

Каждая функция отвечает только за свою часть логики.

━━━━━━━━━━━
<b>ЧАСТЬ 9. Частые ошибки новичков</b>

<b>❌ Забыли вызвать функцию</b>

<pre><code class="language-python">def hello():
    print("Привет!")</code></pre>

Ничего не произойдёт.

Потому что функция только создана.

Нужен вызов:

<pre><code class="language-python">hello()</code></pre>

━━━━━━━━━━━

<b>❌ Забыли скобки при вызове</b>

<pre><code class="language-python">def hello():
    print("Привет!")

hello</code></pre>

Функция не выполнится.

Правильно:

<pre><code class="language-python">hello()</code></pre>

━━━━━━━━━━━

<b>❌ Ошибка в отступах</b>

<pre><code class="language-python">def hello():
print("Привет!")</code></pre>

Будет ошибка:

<pre><code>IndentationError</code></pre>

Нужен отступ:

<pre><code class="language-python">def hello():
    print("Привет!")</code></pre>

━━━━━━━━━━━

<b>❌ Путают print и return</b>

Очень распространённая ошибка.

Если результат нужен дальше в программе — используйте return.

Если нужно просто показать информацию пользователю — используйте print.

━━━━━━━━━━━

<b>❌ Пытаются использовать переменную, которой нет</b>

<pre><code class="language-python">def get_number():
    return x</code></pre>

Если переменная x не существует, возникнет ошибка:

<pre><code>NameError</code></pre>

<b>ЧТО ВАЖНО ЗАПОМНИТЬ</b>
• функция — это именованный блок кода
• функции создаются через <code>def</code>
• функция не выполняется сама по себе
• для выполнения нужен вызов через <code>()</code>
• одну функцию можно вызывать много раз
• <code>print()</code> выводит данные на экран
• <code>return</code> возвращает результат программе
• функции помогают делать код короче и понятнее
• большие программы строятся из множества функций
