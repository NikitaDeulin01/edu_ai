<b>🧩 Урок 4.3. Область видимости и практическое применение функций</b>

<i>Теперь вы умеете создавать функции, передавать в них аргументы и возвращать результат через return. Осталось понять, как функции работают с переменными и почему они помогают строить большие программы.</i>

━━━━━━━━━━━
<b>ЧАСТЬ 1. Что такое область видимости</b>

Когда программа становится большой, появляется много переменных.

Важно понимать:

<b>где именно существует переменная и откуда её можно использовать.</b>

Это называется <b>областью видимости</b>.

Пример:

<pre><code class="language-python">name = "Алекс"</code></pre>

Сейчас переменная доступна во всей программе.

Но если переменная создаётся внутри функции — ситуация меняется.

━━━━━━━━━━━
<b>ЧАСТЬ 2. Локальные переменные</b>

Переменная, созданная внутри функции, называется локальной.

Пример:

<pre><code class="language-python">def hello():
    name = "Алекс"
    print(name)

hello()</code></pre>

Результат:

<pre><code>Алекс</code></pre>

Переменная существует только внутри функции.

━━━━━━━━━━━

Попробуем обратиться к ней снаружи:

<pre><code class="language-python">def hello():
    name = "Алекс"

hello()

print(name)</code></pre>

Получим ошибку:

<pre><code>NameError</code></pre>

Потому что переменная уничтожилась после завершения функции.

━━━━━━━━━━━
<b>ЧАСТЬ 3. Глобальные переменные</b>

Переменная, созданная вне функций, называется глобальной.

Пример:

<pre><code class="language-python">name = "Алекс"

def hello():
    print(name)

hello()</code></pre>

Результат:

<pre><code>Алекс</code></pre>

Функция может читать глобальные переменные.

━━━━━━━━━━━
<b>Глобальные переменные — осторожно</b>

Хотя использовать их можно, злоупотреблять ими не стоит.

Плохой пример:

<pre><code class="language-python">score = 0

def add_points():
    global score
    score += 10</code></pre>

Код начинает зависеть от внешнего состояния.

В больших проектах это часто приводит к ошибкам.

━━━━━━━━━━━
<b>Правило хорошего стиля</b>

Лучше передавать данные через параметры функции.

Вместо:

<pre><code class="language-python">score = 10

def show_score():
    print(score)</code></pre>

лучше:

<pre><code class="language-python">def show_score(score):
    print(score)</code></pre>

Так функция становится независимой.

━━━━━━━━━━━
<b>ЧАСТЬ 4. Изоляция функций</b>

Каждая функция должна выполнять одну конкретную задачу.

Плохой пример:

<pre><code class="language-python">def process_user():
    print("Авторизация")
    print("Загрузка данных")
    print("Отправка письма")
    print("Создание отчёта")</code></pre>

Слишком много обязанностей.

━━━━━━━━━━━

Лучше разделить:

<pre><code class="language-python">def login():
    print("Авторизация")

def load_data():
    print("Загрузка данных")

def send_email():
    print("Отправка письма")</code></pre>

Теперь каждая функция отвечает только за своё действие.

━━━━━━━━━━━
<b>ЧАСТЬ 5. Почему большие программы строятся из функций</b>

Представьте Telegram-бота.

Когда пользователь нажимает кнопку:

<pre><code class="language-python">"Начать обучение"</code></pre>

бот выполняет множество действий.

Например:

<pre><code class="language-python">load_user()
build_plan()
show_lessons()
save_progress()</code></pre>

Каждое действие находится в отдельной функции.

Такой код:

• проще читать

• проще тестировать

• проще исправлять

• проще расширять

━━━━━━━━━━━
<b>ЧАСТЬ 6. Повторное использование функций</b>

Функция создаётся один раз.

Использоваться может сотни раз.

Пример:

<pre><code class="language-python">def square(x):
    return x * x</code></pre>

Использование:

<pre><code class="language-python">print(square(2))
print(square(5))
print(square(10))</code></pre>

Результат:

<pre><code>4
25
100</code></pre>

Один код обслуживает множество ситуаций.

━━━━━━━━━━━
<b>ЧАСТЬ 7. Цепочки функций</b>

Функции могут вызывать друг друга.

Пример:

<pre><code class="language-python">def get_name():
    return "Алекс"

def greet():
    name = get_name()
    print("Привет,", name)

greet()</code></pre>

Результат:

<pre><code>Привет, Алекс</code></pre>

Это обычная практика в реальных проектах.

━━━━━━━━━━━
<b>ЧАСТЬ 8. Разбиение большой задачи</b>

Допустим, нужно вычислить итоговую стоимость заказа.

Новички часто пишут всё в одном месте:

<pre><code class="language-python">price = 1000
discount = 100

result = price - discount

print(result)</code></pre>

Пока код маленький — проблем нет.

Но если логика усложняется, лучше использовать функции.

━━━━━━━━━━━

Пример:

<pre><code class="language-python">def calculate_discount(price):
    return 100

def calculate_total(price):
    discount = calculate_discount(price)
    return price - discount

print(calculate_total(1000))</code></pre>

Так код становится гораздо понятнее.

━━━━━━━━━━━
<b>ЧАСТЬ 9. Мини-проект через функции</b>

Создадим простую программу калькулятора.

Шаг 1:

<pre><code class="language-python">def add(a, b):
    return a + b</code></pre>

Шаг 2:

<pre><code class="language-python">def multiply(a, b):
    return a * b</code></pre>

Использование:

<pre><code class="language-python">print(add(5, 3))
print(multiply(5, 3))</code></pre>

Результат:

<pre><code>8
15</code></pre>

Каждая функция отвечает за свою операцию.

━━━━━━━━━━━
<b>ЧАСТЬ 10. Частые ошибки новичков</b>

<b>❌ Пытаются использовать локальную переменную снаружи функции</b>

<pre><code class="language-python">def test():
    x = 10

print(x)</code></pre>

Ошибка:

<pre><code>NameError</code></pre>

━━━━━━━━━━━

<b>❌ Злоупотребляют глобальными переменными</b>

<pre><code class="language-python">score = 0
balance = 100
user = "Alex"
...</code></pre>

Через некоторое время становится сложно понять, кто меняет эти данные.

━━━━━━━━━━━

<b>❌ Делают одну огромную функцию</b>

<pre><code class="language-python">def do_everything():
    ...</code></pre>

Такие функции трудно читать и поддерживать.

Лучше несколько маленьких функций.

━━━━━━━━━━━

<b>❌ Дублируют код вместо функции</b>

Плохой вариант:

<pre><code class="language-python">print("Привет")
print("Привет")
print("Привет")</code></pre>

Лучше:

<pre><code class="language-python">def hello():
    print("Привет")

hello()
hello()
hello()</code></pre>

━━━━━━━━━━━
<b>ЧТО ВАЖНО ЗАПОМНИТЬ</b>
• переменные внутри функции называются локальными
• локальные переменные существуют только во время работы функции
• переменные вне функций называются глобальными
• функции могут использовать другие функции
• большие программы строятся из множества небольших функций
• каждая функция должна выполнять одну задачу
• функции помогают избегать повторения кода
• хороший код состоит из маленьких понятных функций
• функции — один из главных инструментов программиста
